01 / Product

The brokerless guarantee

Cross-chain XRP trades normally make one side move first. Cinder replaces that trust gap with escrow, competition, collateral and objective proof.

A buyer locks USDT0 and commits an XRPL destination. Independent makers lock FXRP bonds and submit sealed prices. The lowest revealed quote inside the live FTSOv2 band wins. If XRP arrives, FDC releases USDT0 and returns the bond. If it does not, FDC refunds the buyer and slashes the bond.

Flare is the product infrastructure.

Coston2 owns the state and funds, FTSOv2 defines a fair execution band, FXRP supplies native performance collateral, and FDC resolves both cross-chain outcomes without a Cinder operator.

02 / Architecture

Five modules with narrow responsibilities

01 / ESCROWUSDT0 custody

Holds buyer funds and applies the final outcome.

02 / MATCHERSealed auction

Domain-separated commit and reveal competition.

03 / BONDSFXRP vault

Locks, returns or slashes maker collateral.

04 / PRICEFTSOv2 guard

Rejects quotes outside a 1.25% live band.

05 / PROOFDual FDC adapter

Validates payment and referenced non-existence.

Matcher boundary

The guaranteed escrow calls a separate matcher contract. Commit-reveal V2 is the deployed, reliable implementation. An FCC matcher can later implement the same role without rewriting escrow, bond or FDC logic. FCC is not claimed as live because its local runtime was unavailable.

03 / Testnet guide

Run and verify the demo

  1. Open Live proofs. Select Payment delivered or Payment missed.
  2. Click Verify receipts live. The page reads Coston2 receipts directly.
  3. Open the hashes. Inspect USDT0 escrow, maker competition, FDC request and resolution.
  4. Inspect XRPL. The success path links to the validated 2 XRP transaction with the RFQ payment reference.
  5. Create your own hash. In the isolated sandbox, connect a test wallet, claim dcUSD and create a fresh RFQ.

Asset separation: the guaranteed reference uses official Coston2 USDT0 and FXRP. The public sandbox uses an isolated one-claim dcUSD token so judges can transact without controlled maker keys or scarce FXRP faucet balances.

06 / Evidence of new work

Built during the program

  • USDT0-only guaranteed escrow with independent matcher and bond interfaces.
  • FXRP bond vault with one-time controller binding, release and slash accounting.
  • Two-maker domain-separated commit-reveal matcher V2.
  • Dual FDC adapter for Payment and ReferencedPaymentNonexistence proofs.
  • Checkpointable proof keeper that survives RPC resets without duplicating requests.
  • Automated real XRPL payment path and both Coston2 resolution paths.
  • Public proof console plus isolated wallet transaction sandbox.
